在Wiki.js中使用ElasticSearch搜索

Posted on 2021-04-12

首先还是搭建完成Wiki.js，然后搭建ElasticSearch，方法如下：

搭建ES

  • 使用docker pull elasticsearch:7.12.0命令拉取ES，注意要加上版本号，否则可能报错
  • 使用
    docker save -o elasticsearch:7.12.0.tar elasticsearch:7.12.0导出为tar镜像，并导出到离线的机器上，如果没有此需求可以不进行这个操作
  • 在离线机器上使用docker load -i elasticsearch:7.12.0.tar命令导入镜像
  • 启动ESdocker run -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" --name es --restart unless-stopped elasticsearch:7.12.0
  • 添加防火墙规则

    连接ES

  • 登录Wiki.js，进入设置->搜索引擎，选择ElasticSearch
  • Version选择7.x，Host输入http://172.17.0.1:9200，其他的不需要进行修改
  • 点击右上角的应用，再点击右上角的重建索引

这样支持了中文分词搜索，但是亲测搜索效果不佳，很多词汇搜不出来，不知道有无优化余地。

文件夹中的二进制序列搜索

Posted on 2021-04-08

需求：需要在一个文件夹中查找特定的二进制串，例如0x11,0x12,0x13,0x14，经测试可使用grepWin工具

  • Search in 文件夹路径
  • Search for 填写查找文件的正则，例如\x11\x12\x13\x14
  • 勾选Treat files as binary
  • 点击Search即可

缺点是仅查找出来了文件，未写明第几行，还需要UltraEdit之类的工具二次查找。

grepWin工具网站：https://tools.stefankueng.com/

Hibernate、EclipseLink和OpenJPA等ORM框架的区别

Posted on 2021-03-23

把了解到的整理如下：
JDBC是Java连接数据库的方法

iBatis/Mybatis是一种半自动化的ORM框架

  1. Hibernate出现了，Hibernate是冬眠的意思，可能取义“懒人框架”
  2. J2EE团队要求Hibernate在Hibernate的基础上制定了JPA规范，JPA在功能上是Hibernate的子集
  3. Spring团队对JPA进行了封装，产生了Spring Data JPA

也即，JPA是ORM规范，Hibernate实现了JPA规范，同样，EclipseLink、TopLink和OpenJPA也实现了这些规范。

Spring开发中使用的findById(String id) 应该是Spring Data 的特性。

a security risk caused by a 301 error from missing trailing slash in nginx

Posted on 2021-03-23

Our security qa report a risk that ,if you visit a url without a trailing slash ,such as “http://192.168.0.100:7000/home" will cause a 301 error from nginx ,and if you edit the host field in request and resend using firefox debugger tool you may get a reponse with Location field generated from your input host field ,meanwhile a 301 reponse.

So ,what will happen :

  • a hacker edit the request putting a malicious url in the Host field.
  • nginx will redirect it ,sending a 301 reponse with Location generated from Host
  • the browser get the 301 reponse and redirect to the malicious url
